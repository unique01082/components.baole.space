## Contributing

When creating new component guidelines, follow this structure:

1. **Purpose** - What the component does
2. **When to Use** - Appropriate use cases
3. **What NOT to Use For** - Alternatives
4. **Installation** - Import statement
5. **Props API** - Complete props table
6. **Examples** - Basic, controlled, advanced
7. **Do's and Don'ts** - Best practices
8. **Common Patterns** - Real-world examples
9. **Related Components** - Cross-references

See [button.md](./button.md) as the gold standard example.
